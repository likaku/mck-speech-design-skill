## 🔥 核心价值

> **一句话**：你给我 PPT（或任何素材），我给你一份专业演讲稿 + 写好备注的 PPT + 排版好的 Word——全程 AI 协作，十几分钟搞定。

### 它解决什么问题？

每个人都经历过这种崩溃：

> PPT 做好了，但演讲稿还没写。一页一页地想怎么开头、怎么过渡、数据怎么解读……
>
> 写完之后，还要**手动粘贴到每页 PPT 的备注栏**里。
>
> **一页 15 分钟，30 页就是一整个下午。**

这个 Skill 把这个痛苦压缩到 **15 分钟**。

### 效率对比


## 🔄 工作流程

一共 **4 个阶段**，像流水线一样顺畅推进：

```
                    ┌──────────────────────────────────────────────┐
                    │          McKinsey Speech Design Skill        │
                    └──────────────────────────────────────────────┘

  ╔═══════════════╗     ╔═══════════════╗     ╔═══════════════╗     ╔═══════════════╗
  ║  📋 Stage 1   ║      ║  🏗️ Stage 2   ║     ║  ✍️ Stage 3   ║     ║  💉 Stage 4   ║
  ║   Briefing    ║ ──▶ ║ Architecture  ║ ──▶ ║   Drafting    ║ ──▶ ║  Injection    ║
  ╚═══════════════╝     ╚═══════════════╝     ╚═══════════════╝     ╚═══════════════╝
         │                      │                      │                      │
    了解你是谁            金字塔原理              逐页写完整稿           自动写入 PPT
    听众是谁              SCR 框架               融入风格参考           自动导出 Word
    场景和目标            时间分配               过渡句 + 自问自答       两份交付物
    选择风格参考          确认结构                审稿 + 迭代            一键完成
```

### Stage 1: Briefing — 搞清楚「你要什么」

AI 会像一个专业的麦肯锡咨询顾问一样，通过 3-5 轮对话收集关键信息：

| 优先级 | 信息 | 为什么重要 |
|:------:|------|-----------|
| P1 | 素材（PPT/文档/截图） | 内容基础 |
| P1 | 演讲语言 + 时长 | 决定字数和节奏 |
| P1 | 你的身份 + 核心目标 | 定调性 |
| P2 | 听众画像 + 关系深浅 | 调内容深度 |
| P2 | 场景类型（Keynote / 汇报 / Pitch） | 选结构模式 |
| P2 | **🎤 演讲风格参考**（27 选 1） | 注入灵魂 |
| P3 | 必须讲/绝对不能讲的 | 防踩坑 |

### Stage 2: Architecture — 搭建「骨架」

基于**麦肯锡金字塔原理**（Barbara Minto）设计演讲结构：

- **结论先行**：开场就告诉听众你的核心观点
- **SCR 开场框架**：Situation（共识）→ Complication（冲突）→ Resolution（你的答案）
- **MECE 分组**：支撑论点互相独立、完全穷尽
- **精确到分钟的时间表**：按 130-150 字/分钟分配每个章节

这一步会输出结构方案让你确认，通过了才动笔。

### Stage 3: Drafting — 写出「有灵魂的稿子」

不是那种干巴巴的 AI 八股文。这个阶段的写作遵循这些原则：

- **角色代入**：用你的身份说话，不是 AI 的口吻
- **风格融入**：如果你选了 Elon Musk 风格，会出现 *"What people don't realize is..."* 这样的签名句式
- **自问自答技巧**：提前预判听众心里的疑问并回答，制造互动感
- **过渡句必备**：每个章节之间都有自然的桥梁，不会突然跳 topic
- **具体 > 抽象**：用数据、案例和类比，不说空话

### Stage 4: Injection — 自动「交付」

这一步全自动，不需要你操作：

1. 📊 **PPT 文件**：每页备注栏自动填入对应的演讲稿（只放 Script + Transition，简洁可扫）
2. 📄 **Word 文档**：完整参考版（含时间表、Q&A 预案、关键数据备忘、语调提醒）

> 💡 **设计理念**：PPT 备注是演讲时「瞟一眼」的，要简洁；Word 是准备时「细细看」的，要详尽。两份文档各司其职。

## 🎤 风格参考库

**27 种演讲风格**，覆盖科技圈、商业圈、经典演说家、新生代 KOL——选一个编号，AI 就会把 TA 的节奏、用语习惯和叙事方式融入你的演讲稿。

| # | 分类 | 可选风格 |
|---|------|----------|
| **一** | **科技圈领袖** | 1. Elon Musk · 2. Steve Jobs · 3. Donald Trump · 4. Jeff Bezos |
| **二** | **中国商业领袖** | 5. 马云 · 6. 任正非 · 7. 雷军 · 8. 周鸿祎 · 9. 刘强东 |
| **三** | **阿里系（按职级）** | 10. P7-P8 · 11. P9 · 12. P10 · 13. P11-P12 |
| **四** | **字节系** | 14. 张一鸣 · 15. 字节中高层通用 |
| **五** | **华为系** | 16. 任正非 · 17. 余承东 · 18. 华为中高层通用 |
| **六** | **腾讯系** | 19. 马化腾 · 20. 腾讯中高层通用 |
| **七** | **经典演说家** | 21. 马丁·路德·金 · 22. 奥巴马 · 23. 丘吉尔 |
| **八** | **新生代网红/意见领袖** | 24. 罗振宇 · 25. 俞敏洪 · 26. 李开复 · 27. 罗永浩 |

> 不想用？回复「不需要」跳过。风格参考是**调味料**，不是模板——你的演讲稿首先服务于你自己的身份和目标。

### 风格融入的方式

AI 不是简单地「模仿说话」，而是：

- **借 3-5 个签名句式**（适配改写，不是抄）
- **匹配情绪弧线**（Musk = 平稳基调 → 愿景处突然激动；Jobs = 层层铺垫 → One more thing 揭晓）
- **融入幽默风格**（Musk 的冷幽默、周鸿祎的毒舌直白、罗永浩的自嘲）
- **对齐词汇层级**（Trump = 极简重复；任正非 = 文学/军事隐喻；张一鸣 = 精确分析）
- **强度校准**：如果你的实际身份和选的风格差太远，AI 会自动降低 50% 强度——是**灵感**，不是 cosplay

> *"我有一份 30 页的 PPT，明天要给客户做汇报，帮我写一份演讲稿并填入 PPT 备注"*

## ⚙️ 脚本说明

### inject_notes.py — PPT 备注注入

```bash
python scripts/inject_notes.py <原始.pptx> <notes.json> [输出.pptx]
```

- 使用 `python-pptx` API 操作，稳定可靠
- 自动创建 `notesSlide`、处理 `Content_Types` 注册
- 不修改原文件，输出到新文件

### speech_to_docx.py — 演讲稿 Word 导出

```bash
python scripts/speech_to_docx.py <speech.md> [输出.docx]
```

- 解析 Markdown 结构，输出格式化 Word
- 跨平台字体自适应（macOS / Windows / Linux）
- 预编译正则，性能优化

## 💡 为什么做这个

说到底，写演讲稿这件事的痛点不是「不会写」，而是「太耗时间」。

你可能是一个非常优秀的演讲者，但每次都要花几个小时去：
- 构思每一页该怎么讲
- 组织逻辑、写过渡句
- 手动粘贴到 PPT 备注栏
- 导出一份 Word 给自己复习

这些都是**可以被自动化的重复劳动**。

这个 Skill 把你从重复劳动中解放出来，让你把时间花在真正重要的事情上——**练习演讲本身**。
