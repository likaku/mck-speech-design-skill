# Speech Structure Patterns

Proven speech architecture patterns. Select and adapt based on event type, duration, and objectives.

## Foundational Principle: The McKinsey Pyramid (Minto Pyramid Principle)

All patterns below should be **layered on top of** the Pyramid Principle:

```
                    ┌─────────────┐
                    │  Key Message │   ← Audience hears this FIRST
                    │  (So What?)  │
                    └──────┬──────┘
               ┌───────────┼───────────┐
          ┌────▼────┐ ┌────▼────┐ ┌────▼────┐
          │ Pillar 1│ │ Pillar 2│ │ Pillar 3│   ← MECE supporting arguments
          └────┬────┘ └────┬────┘ └────┬────┘
           ┌───┴───┐   ┌───┴───┐   ┌───┴───┐
           │Evidence│   │Evidence│   │Evidence│   ← Data, stories, proof points
           └───────┘   └───────┘   └───────┘
```

### How to apply:

1. **Start with the answer**: State your conclusion/recommendation at the top. Don't make the audience wait.
2. **Group arguments MECE**: Mutually Exclusive, Collectively Exhaustive — no overlaps, no gaps.
3. **SCR opening**: Use Situation–Complication–Resolution as the intro frame:
   - **Situation**: Shared context the audience agrees with
   - **Complication**: What changed, what's at stake, what's the tension
   - **Resolution**: Your key message (= the pyramid apex)
4. **Vertical logic**: Each layer answers "why?" or "how?" from the layer above.
5. **Horizontal logic**: Points within a group follow deductive (premise → conclusion) or inductive (facts → pattern → insight) order.

> **Rule**: Every section of the speech should be traceable upward to the apex message. If a section doesn't support the key message, cut it.

## Pattern 1: The Executive Briefing (20-60 min)

Best for: client meetings, partner exchanges, board presentations.

```
Opening (10%)
├── Welcome & introductions
├── Frame expectations ("What today is / is not")
├── Pose 2-3 questions the audience is likely thinking
└── Promise to address them with facts

Body — 3 themed chapters (70%)
├── Chapter 1: Context & Credibility
│   ├── Who you are (beyond the label)
│   ├── Track record / proof points
│   └── Transition: "Now that you know who we are..."
├── Chapter 2: Market/Domain Insights
│   ├── Data-driven observations
│   ├── Counterintuitive findings
│   └── Transition: "So what does this mean for you?"
└── Chapter 3: Solution / Framework / Proposal
    ├── Your approach / framework
    ├── Honest about challenges
    └── What makes this different

Synthesis & Close (20%)
├── Recap via the 3 opening questions
├── 3 key observations (not promises)
├── Call to action / next steps
└── Open discussion
```

## Pattern 2: The Keynote / Thought Leadership (15-45 min)

Best for: conferences, industry events, public speaking.

```
Hook (5%)
├── Provocative question, statistic, or story
└── "Here's what I want to explore with you today"

Thesis (5%)
├── One-sentence core argument
└── Roadmap: "Three ideas / observations / lessons"

Body — 3 ideas (70%)
├── Idea 1: Setup → Evidence → Insight → Transition
├── Idea 2: Setup → Evidence → Insight → Transition
└── Idea 3: Setup → Evidence → Insight → Transition

Synthesis (15%)
├── Thread the 3 ideas together
├── Return to opening hook
└── Forward-looking statement

Close (5%)
├── Memorable final line
└── Thank you / Q&A invitation
```

## Pattern 3: The Pitch / Proposal (10-30 min)

Best for: sales meetings, fundraising, partnership proposals.

```
Problem (20%)
├── Describe the pain point — use their language
├── Data showing magnitude
└── "Here's what we've observed..."

Solution (40%)
├── Your approach (not product features)
├── Why this approach works (evidence)
├── Differentiation (honest comparison)
└── Case study / proof point

Proposition (30%)
├── Specific next steps
├── Timeline and resources
├── Risk mitigation
└── ROI framework

Close (10%)
├── Recap problem → solution → proposition
├── Clear ask
└── Open for questions
```

## Pattern 4: The Workshop / Interactive Session (60-120 min)

Best for: training, co-creation sessions, strategy workshops.

```
Framing (10%)
├── Today's objective
├── Ground rules
└── Agenda overview

Input Phase (30%)
├── Present framework / content / data
├── Check understanding along the way
└── Pause for questions

Interactive Phase (40%)
├── Exercise / discussion / breakout
├── Facilitate, don't lecture
└── Synthesize group outputs

Wrap-up (20%)
├── Key takeaways
├── Action items
├── Next steps
└── Feedback collection
```

## Pacing Guidelines

| Duration | Words (approx) | Slides (approx) | Sections |
|----------|----------------|------------------|----------|
| 10 min | 1,200-1,500 | 5-8 | 2-3 |
| 20 min | 2,400-3,000 | 10-15 | 3-4 |
| 30 min | 3,600-4,500 | 15-22 | 4-5 |
| 45 min | 5,400-6,500 | 22-35 | 5-7 |
| 60 min | 7,000-8,500 | 30-42 | 6-8 |

Rule of thumb: ~130-150 spoken words per minute. Leave 15-20% buffer for pauses, emphasis, and audience reactions.

## Per-Slide Timing Formula

For a given number of slides and total time:
- `time_per_slide = total_minutes / num_slides`
- Chapter dividers: 0 min (transition only)
- Data-heavy slides: 1.5-2x average
- Summary slides: 0.5-1x average

## Transition Techniques

1. **Question bridge**: "So the next question is..." / "You might be wondering..."
2. **Callback bridge**: "Remember when I mentioned X? Here's why it matters."
3. **Contrast bridge**: "That was the theory. Now let's look at reality."
4. **Preview bridge**: "There are three parts to this. We've covered the first..."
5. **Story bridge**: "Let me illustrate with a specific example."
